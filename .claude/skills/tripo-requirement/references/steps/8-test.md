# 步骤 8：测试验收

## 前置条件

- PR 已创建
- 执行表状态 = "测试中" 或 PR 已合并

## 做什么

1. **等待用户审查 PR**

2. **用户合并 PR 后**：
   - 提议状态变更为"测试中"

3. **配合测试**：
   - 修复发现的 Bug
   - 更新 STATUS.md

4. **测试通过后**：
   - 等待发布上线

## 如何定义完成

- [ ] PR 已合并
- [ ] 测试验收通过
- [ ] 发现的 Bug 已修复
- [ ] STATUS.md 已更新

## 状态变更

| 节点 | 执行表状态 | 需求池状态 |
|------|-----------|-----------|
| PR 合并 | 测试中 | 验收/提测中 |
| 测试通过 | 测试中 | 验收/提测中 |
