# 步骤 9：用户验收

## 前置条件

- Step 8 已完成（自动化闭环已通过）
- 用户已收到通知

## 做什么

1. **等待用户 Review PR**

2. **用户确认后**：
   - 合并 PR
   - 提议状态变更为"提测中"

3. **配合验收测试**：
   - 修复发现的 Bug
   - 更新 STATUS.md

4. **验收通过后**：
   - 进入 Step 10 发布上线

## 如何定义完成

- [ ] PR 已合并
- [ ] 用户验收通过
- [ ] 发现的 Bug 已修复
- [ ] STATUS.md 已更新

## 状态变更

| 节点 | 执行表状态 | 需求池状态 |
|------|-----------|-----------|
| PR 合并 | 测试中 | 提测中 |
| 验收通过 | 测试中 | 提测中 |
