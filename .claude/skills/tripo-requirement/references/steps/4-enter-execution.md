# 步骤 4：进入执行中需求表

## 前置条件

- 需求池状态 = "定容确认"（用户已确认容量）
- 执行中需求表中无此需求的记录

## 做什么

1. **在执行中需求表创建记录**：
   - Table ID: `tblxLMQ8Ih5Gs5oM`
   - Base Token: `HMvbbjDHOaHyc6sZny6cMRT8n8b`

2. **填写字段**：
   - 需求描述
   - 产品Owner、研发Owner、测试Owner
   - 状态 = "评审中" 或 "研发中"
   - 各阶段进度初始值（未启动）
   - 计划提测时间 = 启动时间 + 5 个工作日（粗估，Step 5 技评后会根据工作量重新推算）

3. **更新需求池状态**：
   - 需求池状态 → "开发/交付中"

4. **更新 STATUS.md**
