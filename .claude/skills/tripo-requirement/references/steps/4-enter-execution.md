# 步骤 4：进入执行中需求表

## 前置条件

- 需求池状态 = "定容确认"（用户已确认容量）
- 执行中需求表中无此需求的记录

## 做什么

1. **在执行中需求表创建记录**（→ tripo-tables，字段详情见 fields-execution.md）：
   - 状态 = "评审中"，各阶段进度 = 未启动
   - 计划提测时间 = 启动时间 + 5 工作日（粗估，Step 5 技评后重算）

2. **更新需求池状态** → "开发/交付中"（→ tripo-tables）

3. **记录状态变更**（→ tripo-task-dirs）

## 如何定义完成

- [ ] 执行中需求表已创建记录
- [ ] 需求池状态已更新为"开发/交付中"
- [ ] STATUS.md 已更新
