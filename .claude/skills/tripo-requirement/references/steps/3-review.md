# 步骤 3：需求评审

## 前置条件

- 需求已录入需求池，拥有 record-id
- 需求状态 = "未启动"
- 任务目录下无 review.md

## 做什么

1. **深入理解需求**：
   - 阅读需求描述、PRD 文档
   - 理解业务背景和目标
   - 识别核心功能和边界

2. **输出评审文档**：
   - 创建 `review.md`
   - 内容应包含：
     - 需求来源（ID、描述、Owner、来源文档）
     - 需求范围（核心功能、边界条件、不包含）
     - 验收标准
     - 技术约束
     - 问题与风险
     - 评审结论

3. **更新 STATUS.md**：
   - 添加"需求评审"状态记录

4. **提议状态变更**：
   - 需求状态 → "定容确认"

5. **飞书通知用户**（参见 [notification.md](../notification.md) 节点 1）：
   ```bash
   lark-cli im +messages-send \
     --user-id ou_8adc8aca7ad728142eb6669e5b13fb52 \
     --text $'[需求评审完成]\n需求: <需求名称>\n状态: 已输出评审文档，提议变更为"定容确认"\n操作: 请在 Claude Code 中确认容量，或提出修改意见'
   ```
   通知后**暂停**，等待用户确认。

6. **等待用户确认容量**

## 如何定义完成

- [ ] review.md 已输出
- [ ] STATUS.md 已更新
- [ ] 已提议状态变更为"定容确认"
- [ ] 用户已确认容量（进入下一步）
