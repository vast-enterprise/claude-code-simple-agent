# 步骤 3：需求评审

## 前置条件

- 需求已录入需求池，拥有 record-id
- 需求状态 = "未启动"
- 任务目录下无 review.md

## 做什么

1. **深入理解需求**：
   - 阅读需求描述、PRD 文档
   - 理解业务背景和目标
   - 识别核心功能和边界

2. **输出评审文档**：
   - 创建 `review.md`
   - 参见 [templates/review.md](../templates/review.md)

3. **更新 STATUS.md**：
   - 添加"需求评审"状态记录

4. **提议状态变更**：
   - 需求状态 → "定容确认"

5. **等待用户确认容量**

## 如何定义完成

- [ ] review.md 已输出
- [ ] STATUS.md 已更新
- [ ] 已提议状态变更为"定容确认"
- [ ] 用户已确认容量（进入下一步）

## 评审文档要点

- 核心功能明确
- 验收标准清晰
- 边界条件定义
- 风险点识别
