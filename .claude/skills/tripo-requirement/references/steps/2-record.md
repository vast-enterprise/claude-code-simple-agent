# 步骤 2：归类与录入

## 前置条件

- 任务目录已创建（STATUS.md 已初始化）
- 需求池中无此需求的记录

## 做什么

1. **判断需求类型**：
   - 产品需求 → 产品需求池
   - 技术需求 → 技术需求池

2. **录入需求池**（→ tripo-tables，字段详情见对应表的 references）：
   - 一句话描述需求
   - 需求详细描述（可附文档）
   - 需求Owner、研发Owner
   - 需求提出日期
   - 需求池分类
   - 绝对优先级
   - 需求状态 = "未启动"

3. **更新任务目录**（→ tripo-task-dirs）：
   - 重命名目录（临时 ID → 实际 ID）
   - 更新 STATUS.md

## 如何定义完成

- [ ] 需求已录入对应表格
- [ ] 已获取 record-id
- [ ] 任务目录名已更新（如有临时 ID）
- [ ] STATUS.md 已更新
