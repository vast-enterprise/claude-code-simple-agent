# 步骤 10：发布上线

## 前置条件

- 测试验收通过
- 执行表状态 = "测试中"，需求池状态 = "验收/提测中"

## 做什么

1. **飞书通知用户**（→ tripo-tables，notification.md 节点 4）：
   通知后**暂停**，等待用户确认。

2. **等待用户发布上线**

3. **确认上线完成**：
   - 用户告知已上线
   - 或通过其他方式确认

4. **提议状态变更**（→ tripo-tables）：
   - 执行表状态 → "完成"
   - 需求池状态 → "已完成"
   - 实际交付时间 → 当前日期
   - 发车路径选择参见 `tripo-tables` skill 的 `release-flow.md`

5. **更新 STATUS.md**（→ tripo-task-dirs）：
   - 状态标记为 ✅ 已完成

6. **归档任务目录**（→ tripo-task-dirs）

7. **（可选）清理 worktree**（→ tripo-worktree）

## 如何定义完成

- [ ] 用户确认已上线
- [ ] 已提议状态为"完成"
- [ ] 实际交付时间已更新
- [ ] STATUS.md 已更新为完成状态
- [ ] 任务目录已归档

## 最终状态

| 表格 | 状态字段 | 值 |
|------|---------|-----|
| 需求池 | 需求状态 | 已完成 |
| 执行表 | 状态 | 完成 |
