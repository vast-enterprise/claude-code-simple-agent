# 步骤 5：技术评审

## 前置条件

- 执行表记录已存在
- 执行表"技术评审"字段 ≠ "完成"
- 任务目录下无 technical-solution.md

## 做什么

1. **分析技术方案**：
   - 评估多种实现方案
   - 考虑性能、可维护性、风险

2. **输出技术方案文档**：
   - 创建 `technical-solution.md`
   - 参见 [templates/technical-solution.md](../templates/technical-solution.md)

3. **更新 STATUS.md**

4. **提议状态变更**：
   - 执行表"技术评审"字段 → "完成"

## 如何定义完成

- [ ] technical-solution.md 已输出
- [ ] STATUS.md 已更新
- [ ] 已提议"技术评审"="完成"
- [ ] 用户已确认方案

## 技术方案要点

- 多方案对比
- 推荐方案及理由
- 涉及模块清单
- 工作量评估
- 风险点及应对
