# 步骤 7：创建 PR

## 前置条件

- 代码已通过 lint、typecheck、test
- 尚未创建 PR

## 做什么

1. **提交代码**
2. **创建 PR**
   - 关联需求 ID
   - 关联任务目录
   - 描述变更内容
3. **更新 STATUS.md**

## 如何定义完成

- [ ] PR 已创建
- [ ] STATUS.md 已更新

## PR 格式

```markdown
## 关联需求
- 需求ID: <record-id>
- 任务目录: tasks/2026-04-02_REQ-xxx_xxx/

## 变更内容
- ...
```

## ⚠️ 注意

创建 PR 后**继续 Step 8**，不要通知用户。
