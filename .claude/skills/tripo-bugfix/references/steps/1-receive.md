# 步骤 1：接收 Bug

## 前置条件

- 用户提供了 bug 信息（文字描述、截图、消息转发、线上链接等）

## 做什么

1. **明确三要素**：
   - **现象**：什么功能出了什么问题？
   - **环境**：生产 / staging / 本地？哪个页面？
   - **仓库**：涉及哪个代码仓库？（→ tripo-repos）

2. **信息不足时**：AskUserQuestion 补齐，不要猜测

3. **评估优先级**（P0/P1/P2）

4. **创建任务目录**（→ tripo-task-dirs）：
   - 目录格式：`tasks/<日期>_BUG-<bug简述>/`
   - 初始化 STATUS.md

## 如何定义完成

- [ ] bug 三要素已明确（现象、环境、仓库）
- [ ] 优先级已评估
- [ ] 任务目录已创建，STATUS.md 已初始化
