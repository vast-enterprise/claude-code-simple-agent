# 步骤 5：验证

## 前置条件

- 步骤 4 完成：代码修改完成，本地验证通过

## 做什么

1. **准备测试环境**：
   - 如果 staging 上已有触发 bug 的数据，直接用
   - 如果没有，在 staging 环境创建测试数据

2. **推送分支到远程**：
   - `git push -u origin <branch-name>`

3. **加载 `tripo-test` skill 执行验证**：
   - 步骤 3 中列出的每一个受影响项都要验证
   - 回归检查：修复是否引入新问题？已有功能是否正常？

## 返工

验证不通过时，加载 `tripo-dev` skill 修复代码，修复后重新执行本步骤验证。

## 如何定义完成

- [ ] 分支已推送到远程
- [ ] 每个受影响项都有验证证据
- [ ] 回归检查通过
