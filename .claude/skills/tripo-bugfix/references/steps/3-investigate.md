# 步骤 3：调查 & 定位根因

## 前置条件

- 步骤 2 完成：Bug 已录入管理表

## 做什么

1. **进入涉及的代码仓库**（→ tripo-repos）

2. **加载 `tripo-diagnose` skill 执行调查**

3. **向所有者报告结论**：
   - 用人话解释根因
   - 列出受影响清单
   - 提出修复建议

4. **更新 STATUS.md**（→ tripo-task-dirs）

5. **更新 Bug 表**（→ tripo-tables）：
   - 进度：Open → In progress
   - 更新 `复现步骤` 字段（补充根因分析）

## 如何定义完成

- [ ] 根因已定位到具体代码位置
- [ ] 受影响范围已列出完整清单
- [ ] 已向所有者报告结论
- [ ] Bug 表已更新为 In progress
