# 步骤 2：录入 Bug 管理表

## 前置条件

- 步骤 1 完成：bug 现象、涉及仓库、优先级已明确

## 做什么

1. **录入前：向用户确认关键字段**（→ tripo-tables 的「录入前确认机制」）

   关键字段清单：查 `fields-bug.md` 中标注【录入关键字段，需确认】的字段。本流程关键字段包括：`提出人`（真实提出人）、`指派人`、`优先级`、`类型`。

   **Bug 表特别规则**（详见 tripo-tables「Bug 表特别规则」）：
   - `创建人`（`fld1mMy65N`）是 `created_by` 系统字段，代录入场景会错位
   - 真实提出人必须在 `备注`（`fldemHmVhH`）字段用**飞书原生 `@` 人语法**显式标注
   - 通知模块（tripo-notify）据此读备注取 open_id

2. **在 Bug 管理表创建记录**（→ tripo-tables，Bug 表）：
   - 必填：bug 描述、优先级、类型（bug/产品体验）、进度=Open
   - 必填（备注）：飞书原生 `@` 真实提出人
   - 指派人：若已明确则填，否则留空（后续 diagnose 定位后补）
   - 选填：复现步骤、发现阶段、关联需求、taskID/traceID

3. **记录 record_id**：用于后续状态更新

## 如何定义完成

- [ ] 录入前已按 tripo-tables 的「录入前确认机制」向用户确认关键字段
- [ ] Bug 管理表中已创建记录
- [ ] 备注字段已用飞书原生 `@` 语法显式标注真实提出人
- [ ] **录入后校验**：用 `record-get` 读回备注字段，确认富文本结构中包含至少一个 mention 元素；没有则补录
- [ ] record_id 已记录
- [ ] 进度为 Open
