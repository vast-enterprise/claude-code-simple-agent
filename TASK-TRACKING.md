# 任务进度跟踪规范

> 本文档定义任务进度跟踪目录的结构和更新规则。

## 目录结构

```
tasks/
├── 2026-04-02_REQ-recXxx_用户画像分析优化/
│   ├── STATUS.md                    # 状态追踪记录（实时更新）
│   ├── review.md                    # 需求评审文档
│   ├── technical-solution.md        # 技术方案文档
│   └── notes/                       # 其他笔记（可选）
│       └── meeting-2026-04-03.md
│
├── 2026-04-05_REQ-recYyy_支付流程重构/
│   └── ...
│
└── 2026-04-10_BUG-recZzz_首页加载问题修复/
    └── ...
```

## 目录命名规范

**格式**: `{日期}_{类型}-{ID}_{简述}`

| 部分 | 说明 | 示例 |
|------|------|------|
| 日期 | 需求启动日期 | `2026-04-02` |
| 类型 | REQ（需求）/ BUG（Bug修复） | `REQ` |
| ID | 表格 record-id 或编号 | `recXxx` |
| 简述 | 中文简要描述（不超过20字） | `用户画像分析优化` |

**示例**:
- `2026-04-02_REQ-recAbc123_博客系统SEO优化`
- `2026-04-05_BUG-recDef456_首页白屏问题修复`

## 状态标记说明

| 标记 | 含义 |
|------|------|
| ⏳ | 未启动 |
| 🔄 | 进行中 |
| ✅ | 已完成 |
| ⚠️ | 有风险 |
| ❌ | 已暂停 |

## 更新规则

### 必须更新 STATUS.md 的时机

1. **阶段状态变更时**
   - 开始某阶段：更新为 🔄
   - 完成某阶段：更新为 ✅
   - 遇到风险：更新为 ⚠️
   - 暂停开发：更新为 ❌

2. **PR 提交时**
   - 记录 PR 编号
   - 更新阶段状态

3. **需求变更时**
   - 在备注中记录变更内容
   - 更新预期交付时间（如有变化）

### 更新命令

```bash
# 在任务目录下直接编辑 STATUS.md
# 无需命令，直接使用 Edit 工具更新
```

## 目录创建命令

当接收到新任务时，Agent 应：

1. 创建任务目录：
```bash
mkdir -p tasks/2026-04-02_REQ-recXxx_简述/notes
```

2. 初始化 STATUS.md：
- 填写基本信息
- 添加第一条状态历史记录

3. 后续根据进度实时更新 STATUS.md
